# Community Post Templates (KR/EN)

원칙: 커뮤니티는 “자동 업로드”보다 **수동 업로드 + 댓글 대응**이 성과에 결정적이다.  
자동화는 초안 생성/버전 관리/채널별 포맷 변환까지만 추천.

## 1) 국내(Disquiet/OKKY/Velog) — 문제→해결→배운점 톤

### 제목(예시)
- “만다라트를 ‘계획’이 아니라 ‘실천’으로 바꾸는 앱을 만들었습니다(피드백 부탁드립니다)”
- “오타니 만다라트를 GTD로 변환하는 앱을 만들었는데, UX 조언 구합니다”

### 본문 골격
1. 배경/문제: “만다라트는 좋은데 실행이 어렵더라”
2. 해결: “그리드를 ‘오늘의 체크리스트’로 자동 변환”
3. 특징 3개(짧게): OCR 입력 / 오늘의 실천 / 주간 리포트
4. 지금 필요한 피드백 2~3개(구체적 질문)
5. 링크(앱스토어/랜딩/피드백 채널)

## 2) 글로벌(Product Hunt/Reddit/HN/X) — value-first 톤

### Reddit (r/productivity 등)
- “I built X because Y problem” + 실제 사용 팁 3개 + “feedback welcome”
- 서브레딧 룰(자기홍보 제한) 준수 필수

### Hacker News (Show HN)
- 빌드 과정에서 배운점/기술 포인트를 1~2개 포함하면 반응이 좋아짐

### X (Twitter)
- 1) Hook 한 줄 2) 스크린샷/짧은 데모 3) CTA 1개
